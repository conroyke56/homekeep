## Cross-Phase Integration

The milestone's coordinating thesis is the 6-branch `computeNextDue` composition:

```
archived → override (10) → smoothed (12) → seasonal-dormant (11) → seasonal-wakeup (11) → OOFT (11) → cycle/anchored-natural
```

LOAD-15 branch matrix (Phase 12 hard gate) proves 21 explicit interactions:
- 6 precedence cases (override > smoothed > seasonal > OOFT > natural)
- 15 pairwise interaction cases covering every meaningful combination

Phases 10-11-12 nest their read-side branches correctly; Phase 13 TCSEM + Phase 17 REBAL reuse Phase 12's `placeNextDue` + `computeHouseholdLoad` primitives; Phase 15 reschedule `reschedule_marker` is consumed by Phase 17's REBAL preservation classifier.

Write-side atomicity: `completeTaskAction` batches {completion write + override consume + OOFT archive + smoothed placement} in a single `pb.createBatch()` — all four ops roll back together on any failure.

Port allocation integrity: 18090..18105 claimed across 16 disposable-PB integration suites, zero collisions.

## Tech Debt Summary

See frontmatter `tech_debt` section for per-phase lists. Two items are production-relevant and warrant near-term attention:

1. **Phase 14: PB 0.37.1 NumberField 0-vs-null quirk for seasonal months.** Dashboard / By Area / Person server components map `active_from_month` via `(t.active_from_month as number) ?? null`, which doesn't catch `0` that PB returns when a NumberField is cleared. Downstream `classifyDormantTasks` treats `0` as "has window" and misclassifies non-seasonal tasks as dormant. Workaround exists in the visual-UAT test spec; production code uncovered. Low risk in practice (users don't clear seasonal months often), but worth a patch before v1.1.0-rc2.

2. **Phase 14: Seasonal-wakeup fires for fresh task with year-round window.** A task with `active_from=1, active_to=12` and no completion triggers the `wasInPriorSeason` branch and returns `nextWindowOpenDate` (Jan 1 next year) instead of natural cadence. Also surfaced by visual UAT. Same severity as #1.

Everything else is info-level (quality nits, cast inconsistencies, redundant fetches). No blockers.
