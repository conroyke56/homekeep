## Executive Summary

v1.2.1 is a patch on top of v1.2.0, motivated entirely by a single Playwright live-smoke test run against the deployed VPS that turned up two production bugs no hermetic test caught. During the fix round the user surfaced two UX concerns and one dead-link that were bundled into the same milestone.

Seven REQ-IDs (PATCH2-01..07) shipped across four phases. All four phases were formalized via GSD — Phases 29-30 retroactively (the underlying fixes landed as inline commits before the milestone was created; SUMMARY.md files in each phase dir point at the commits), Phases 31-32 as new work.

The milestone also hardened the release pipeline itself: the Release workflow now auto-creates GitHub Release objects on tag push (prior workflow shipped images + SBOM + provenance but never the Release object — the Releases page was stuck on v1.0.0 despite v1.1.x and v1.2.0 tags existing). Four missing Releases were backfilled via API before v1.2.1 landed so the page is consistent.

## REQ Coverage

**PATCH2-01** — Auth cookie `Secure` derived from `SITE_URL.startsWith('https://')`, not `NODE_ENV`. Fixes LAN-HTTP production regression (cookie dropped over plain HTTP even though browser requested it). File: `lib/actions/auth.ts` (`isSecureSite()` helper).

**PATCH2-02** — Remember-me checkbox on the login form; `cookieOptions(rememberMe: boolean)` returns a session cookie when unchecked, 14-day persistent when checked. Default checked (matches prior behavior). Files: `lib/actions/auth.ts`, `components/forms/login-form.tsx`.

**PATCH2-03** — Caddyfile explicit allow-list for Next.js API paths before the PB `/api/*` catch-all. `/api/csp-report` now returns 204 through the edge port 3000 instead of 404. File: `docker/Caddyfile`.

**PATCH2-04** — Removed the `<Link href="/deployment">` from `InsecureContextBanner`. Route never existed; Next.js prefetched its RSC payload on hydration, producing 9+ console 404s per HTTP session. Banner informational copy kept. Files: `components/insecure-context-banner.tsx`.

**PATCH2-05** — `PASSWORD_POLICY` env var (and `NEXT_PUBLIC_PASSWORD_POLICY` for client bundle parity). Default `simple` = 8-char floor on signup + reset. Opt-in `strong` = 12-char floor (Phase 23 SEC-06 behavior). Login schema keeps 8-char floor for back-compat. Schemas use `.superRefine()` so the min is read at parse time, not module load — enables per-test `vi.stubEnv` without module-cache juggling. Files: `lib/schemas/auth.ts`, `.env.example`, `docs/deployment-hardening.md` (item 12b).

**PATCH2-06** — Task tap opens the detail sheet by default (iOS-style tap-to-view). Completion reaches through the sheet's Complete button. `TaskRow` gains `primaryTap?: 'complete' | 'detail'`; default is `'detail'` when `onDetail` is provided. `PersonTaskList` passes `primaryTap="complete"` explicitly because its `onDetail` opens a reschedule sheet, not a detail view. Files: `components/task-row.tsx`, `components/task-band.tsx`, `components/person-task-list.tsx`.

**PATCH2-07** — Never-completed tasks bypass the early-completion guard. `shouldWarnEarly` returns `false` when `lastCompletion === null`; the `task.created` fallback was producing false positives on brand-new tasks (common case: "I just added this task and I'm doing it today"). File: `lib/early-completion-guard.ts`.

## Release-Pipeline Follow-Ups (non-REQ)

1. **Auto-create GitHub Release on tag push** — `release.yml` gains `softprops/action-gh-release` step. `prerelease` flag = `contains(tag, '-')`; `make_latest` flag inverted. First tag to auto-create: v1.2.1. Four prior tags (v1.1.0-rc1, v1.1.1-rc1, v1.1.1, v1.2.0) backfilled via API.
2. **CI timeout bump 25 → 45 min** on `lint-test-build` — arm64 cross-build under QEMU was racing the 25-min ceiling after the E2E suite grew past ~25 specs.
3. **NEXT_PUBLIC_ regression check → allow-list** — Phase 1 Pitfall-#3 check banned all `NEXT_PUBLIC_*` to catch leaks; PATCH2-05 introduced an intentional public var. The gate now scans for unexpected public vars against an allow-list (`NEXT_PUBLIC_PASSWORD_POLICY`) — still catches a rogue `NEXT_PUBLIC_API_KEY` regression.
