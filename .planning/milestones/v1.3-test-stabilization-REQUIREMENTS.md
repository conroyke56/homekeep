## REQ-IDs

### TESTFIX-01 — Unskip `homes-areas.spec.ts:41` (area rename race)

**Currently:** `test.skip()` with "Save → redirect → /areas assertion
race" comment.
**Root cause:** Next.js Server Action redirect + DOM update race —
the `page.goto('/h/<id>/areas')` fires before the renamed area row
has rendered.
**Fix:** Replace the immediate `.toBeVisible()` assertion with
`page.waitForResponse(/\/api\/.*areas.*/)` after Save, then
`.toBeVisible({timeout: 5000})`. Estimated: 10-15 min.
**Done when:** test is un-skipped, runs green on CI 3 runs in a row.

### TESTFIX-02 — Unskip `notifications.spec.ts:150` (RHF conditional reveal)

**Currently:** `test.skip()` with "RHF-Controller-wrapped weekly-
summary checkbox → conditional day-select reveal race" comment.
**Root cause:** React Hook Form's `Controller` + `useWatch` fires the
conditional render on a subsequent tick; Playwright's assertion runs
before that tick under CI headless.
**Fix:** Replace `.toBeVisible()` with an explicit
`page.waitForSelector('[data-field=weekly-summary-day]', {timeout:
5000})` or bump the visibility timeout. Estimated: 10-15 min.
**Done when:** test is un-skipped, runs green on CI 3 runs in a row.

### TESTFIX-03 — Fix signup → `/h` redirect race (18 assertions across 12 specs)

**Currently:** intermittent 302-loop failures on the post-signup URL
assertion. Affects every spec that calls the `signup(page, ...)`
helper — inherited flake, not per-test.
**Root cause:** `signupAction` in `lib/actions/auth.ts:142-202` calls
`pb.collection('users').create()` + `authWithPassword()` + cookie set
+ `revalidatePath('/h')` + `redirect('/h')`. Under CI concurrency
(rate-limits disabled, 15+ parallel signups/min), the cache
revalidation races the redirect — browser navigates before auth
middleware has the session, producing either a redirect loop or a
stale 401.
**Fix options:**
- **A (server-level, recommended):** restructure `signupAction` so
  `revalidatePath` and cookie set both complete before `redirect`
  fires. Likely involves `await`-ordering or a `Promise.all`
  restructure. ~30-45 min.
- **B (test-level band-aid):** add explicit
  `page.waitForURL(/\/h$/, {timeout: 15000})` to all 18 assertion
  sites + document in `tests/e2e/helpers.ts`. ~15 min but doesn't
  fix the actual race.
- **C (RSA polling pattern):** introduce a server-settle wait
  endpoint and poll from the form. ~1-2 hours, overkill.
**Pick:** A. Fall back to B only if A breaks something upstream.
**Done when:** 18 assertions pass 10 CI runs consecutively with no
flake re-appearance.

### TESTFIX-04 — Abstract shared test helpers for future flake-resistance

**Add to `tests/e2e/helpers.ts`:**
- `signupAndWait(page, email, pw, name?)` — wraps `signup()` with
  the post-redirect polling pattern from TESTFIX-03; use this as
  the default signup helper so future tests inherit the fix
- `waitForServerAction(page, expectedUrl, timeout?)` — generic
  "wait for Server Action to complete + URL settle" helper
- `waitForRHFConditional(page, triggerSelector, revealSelector,
  timeout?)` — wait for RHF `useWatch` → conditional render,
  prevents future RHF-like flakes from the SKIP-02 class
**Done when:** helpers exist, existing 18 post-signup assertions
migrate to `signupAndWait`, all tests green.

### TESTFIX-05 — CI flakiness budget + detection

**Scope:** a follow-up guard so v1.4 doesn't introduce new flakes
silently. Add to the CI workflow:
- A retry-on-flake policy limited to **2 retries max per test**
  (Playwright native `retries` field on CI-only)
- A workflow step that fails CI if retries were actually consumed
  (surface flake with "flaky: 3/24 tests retried" rather than
  silently passing)
**Done when:** CI reports retry count; zero retries used on a clean
PR would be the default.

### TESTFIX-06 — Planning-doc cleanup sweep

stay clean. Also review the `.planning/phases/` tree for any retro-
active phase summaries that could be consolidated.
**Done when:** one current HANDOFF at repo root level, older ones
archived with a pointer README.
