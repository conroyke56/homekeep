# Phase 39 Plan 01 — Hydration Signal Fix + Un-skip Notifications Tests

Single-wave plan. Three tasks land in one commit; the fix is small
enough that splitting it adds more overhead than it saves.

## Task 1 — Add hydration signal to NotificationPrefsForm

**File:** `components/notification-prefs-form.tsx`

**Change:** replace the SSR-baked `data-notifications-ready="true"`
literal with a `useState`+`useEffect` post-hydration flip.

**Specific edits:**

1. Add `useState` to the React import (already imports `useEffect`,
   `useActionState`, `useRef`):

   ```tsx
   import { useActionState, useEffect, useRef, useState } from 'react';
   ```

2. Inside `NotificationPrefsForm`, after the existing
   `useActionState` hook and before the `useForm` call, add:

   ```tsx
   const [hydrated, setHydrated] = useState(false);
   useEffect(() => {
     setHydrated(true);
   }, []);
   ```

3. Change the Card root attribute (line 100):

   ```tsx
   // before:
   data-notifications-ready="true"
   // after:
   data-notifications-ready={hydrated ? 'true' : 'false'}
   ```

**Acceptance:**

- `data-notifications-ready` reads `"false"` in SSR HTML output
  (verifiable via `curl http://localhost:3000/h/<id>/person | grep
  data-notifications-ready` after a build).
- Reads `"true"` after hydration in browser DevTools.
- No TypeScript errors; no new lint findings.
