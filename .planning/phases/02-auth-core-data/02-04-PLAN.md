<objective>
Build the homes and areas CRUD surface: user can create homes, switch between them (with last-viewed persistence), and fully manage areas including the drag-reorder UI. After this plan, covering 9 requirements (HOME-01..04, AREA-01..05), a user who signed up in 02-03 can create their actual maintenance domain — the home and its areas — and the "Whole Home" auto-create invariant is visible end-to-end.

Purpose: Homes are the root of the data model. Areas are the first user-creatable subdivision and the prerequisite for Task creation (02-05 depends on area records existing). This plan also delivers the first real drag-and-drop UI (@dnd-kit) and the first batched-write server action (`pb.createBatch`) — both templates downstream phases will copy.

Output: Schemas + actions + forms + 3 new route-group pages + 4 small Client Components (HomeSwitcher, SortableAreaList, IconPicker, ColorPicker) + area-palette constants. Two Vitest schema test files + one Playwright E2E covering the home+area happy path.
