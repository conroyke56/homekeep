## Single remaining user-side action to launch the public demo

The `homekeep` user (what Claude runs as) can't write to
`/home/sprout/revproxy/vhosts/`. Run these 3 commands from your
`sprout` shell (or root):

```bash
# 1. Copy the vhost snippet into the shared revproxy
sudo install -m 644 -o sprout -g sprout \
  /srv/homekeep/docker/revproxy-vhosts/homekeep.demo.the-kizz.com.caddy \
  /home/sprout/revproxy/vhosts/homekeep.demo.the-kizz.com.caddy

# 2. Reload revproxy so it picks up the new vhost
docker exec revproxy-caddy-1 caddy reload --config /etc/caddy/Caddyfile

# 3. Bring up the demo stack on 127.0.0.1:3001 (coexists with personal on 3000)
cd /srv/homekeep/docker
docker compose -p homekeep-demo \
  -f docker-compose.yml \
  -f docker-compose.demo.yml \
  -f docker-compose.demo-vps.yml \
  --env-file .env.demo.local \
  up -d

# 4. Verify (give Caddy ~30s to ACME):
curl -sS https://homekeep.demo.the-kizz.com/api/health
# Expect: {"status":"ok","nextjs":"ok","pocketbase":"ok","pbCode":200}
```

Architecture:
- **Personal:** `homekeep` container, port 3000 (existing, untouched)
- **Demo:** `homekeep-demo` container, 127.0.0.1:3001 (new)
- **Edge:** `revproxy-caddy-1` (existing shared Caddy on --network host) routes `homekeep.demo.the-kizz.com` → `127.0.0.1:3001` with Let's Encrypt HTTPS
